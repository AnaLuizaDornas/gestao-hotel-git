Relatório do Projeto - Sistema de Gestão de Reservas de Hotel com Git e GitHub
Organização do Trabalho O desenvolvimento foi realizado individualmente por Ana Luiza Dornas, seguindo uma abordagem baseada em controle de versões com Git e colaboração via GitHub. O projeto foi estruturado em um repositório público (https://github.com/AnaLuizaDornas/gestao-hotel-git), criado conforme as instruções iniciais. Três branches principais foram estabelecidas: feature-clientes, feature-quartos, e feature-reservas, cada uma dedicada a uma funcionalidade específica. O trabalho foi dividido em etapas, com commits frequentes e mensagens descritivas (ex.: "Adiciona função para listar clientes registrados"). As integrações foram realizadas por meio de Pull Requests (#1, #2, #3, #4) no GitHub, revisados e mergidos no ramo main, garantindo um fluxo de trabalho iterativo e documentado.
Problemas e Conflitos Diversos desafios surgiram durante o processo. Inicialmente, o ramo main local ficou desatualizado, com commits de quartos realizados acidentalmente, o que desalinhou feature-quartos. Um erro de importação em reservas.py ocorreu devido à ausência inicial de funções em quartos.py. Um conflito significativo surgiu na função listar_quartos entre feature-reservas e main, devido a modificações divergentes. Além disso, foi necessário tratar o caso de reservas em quartos ocupados e ajustar imports entre módulos para assegurar a interoperabilidade.
Resolução com Git Os problemas foram resolvidos com técnicas avançadas de Git. Os commits acidentais em main foram corrigidos recriando manualmente as alterações em feature-quartos e enviando-as via git push. O erro de importação foi sanado recriando as funções em quartos.py e integrando-as com git commit e git push. O conflito em listar_quartos foi resolvido usando git merge, editando manualmente o arquivo para combinar as versões e finalizando com git commit. A funcionalidade de "Quarto ocupado" foi adicionada em feature-reservas, commitada e mergida. Por fim, a tag v1.0 foi criada com git tag -a v1.0 -m "Versão final do sistema de gestão de hotel" e enviada com git push origin v1.0, marcando a conclusão do projeto.
